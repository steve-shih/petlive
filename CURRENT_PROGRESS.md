# PetLive 專案目前開發進度報告

## 🚀 已完成的核心功能

### 1. 使用者與基本功能
- **會員系統**：Line Login 整合與一般信箱註冊/登入。
- **商城功能**：商品列表展示、商品詳細頁、購物車操作與結帳流程。
- **即時聊天**：支援全局與一對一的 WebSocket 即時聊天功能。
- **賣家後台**：商品上架、商品管理、賣家身分認證（VIP Tier 系統）。

### 2. 進階 Mesh P2P 直播系統 (核心亮點)
我們實作了一套基於 WebRTC (PeerJS) 的**無中心化樹狀 P2P 直播分發系統**，大幅降低伺服器頻寬壓力，並達成以下亮點：
- **自訂拓撲結構**：直播主開播前可進入「進階設定」，自訂：
  - **最大層數 (Max Layers)**：控制網路深度。
  - **直播主乘載量 (Layer 0 Capacity)**：限制直播主直接連線的觀眾數。
  - **觀眾轉發乘載量 (Layer N Capacity)**：限制每個觀眾節點可轉發給多少下層觀眾。
  - 系統會即時試算並顯示**理論最大乘載人數**。
- **動態 Peer ID 核發**：為徹底解決重整頁面造成的 ID 衝突，所有節點均透過 PeerJS 雲端動態取得唯一 ID 後，再向後端 `/join` 報到，由後端 BFS 演算法尋找最適合的父節點。
- **自我修復機制 (Self-Healing)**：當上層節點（或父節點）斷線時，下層觀眾會觸發 `report_dead` API 剔除失效節點，並自動重新向伺服器請求新的父節點來恢復連線。
- **行動端專屬 UX**：
  - **沉浸式全螢幕**：手機端看直播時會自動隱藏多餘 UI，呈現全螢幕觀看體驗。
  - **滑動切換房間**：支援類似抖音的上下左右滑動，快速切換不同直播間。
  - **前後鏡頭切換**：直播主可隨時點擊按鈕無縫切換手機前後鏡頭 (`facingMode`)。
- **防止瀏覽器自動播放封鎖 (Autoplay Blocked)**：若瀏覽器基於隱私限制阻止了影片自動播放，UI 會自動跳出「畫面已暫停 (請點擊畫面播放聲音與影像)」防呆提示，點擊後即可無縫接續播放。

### 3. 基礎建設與部署
- **前端**：Next.js (App Router) + TailwindCSS，支援 RWD 響應式設計。
- **後端**：Python Flask + MongoDB，處理 REST API 與 Mesh 節點追蹤樹 (`MESH_TREES`)。
- **服務管理**：使用 PM2 進行後端與前端的程序守護 (Daemon) 與日誌追蹤。
- **全域設定檔**：將通用參數與設定抽離至 `config.json` 統一管理。

## 🛠️ 待優化與下一步計畫
1. **P2P 穿透率提升**：若未來需支援複雜的 4G/5G 浮動 IP 與嚴格 NAT 環境，建議導入自建的 STUN/TURN Server。
2. **直播留言板與斗內**：可將現有的 WebSocket 聊天室進一步整合進直播間畫面中，實作送禮與跑馬燈特效。
3. **推播通知**：當關注的賣家開播時，透過 PWA 或 Line Notify 發送通知。
